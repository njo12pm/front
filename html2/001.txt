1. 네트워크를 사용하는 이유
   - 자원 공유(프린터, 파일 서버...) -> 지금 누가 사용할 것인가?
   - 통신규약(프로토콜)이 필요하다

2. 간단하게 바라보는 인터넷
    구조           통신규약                   비유

    서비스         HTTP, FTP, SMTP...         버스, 택배...
    인터넷         TCP/IP                     도로
    lan            ethernet

3. IP(Internet Protocol)
   - 인터넷에 연결된 호스트를 구별할 수 있는 주소를 제공
   - IP 주소는 국제적으로 관리
   - 장비를 인터넷에 연결하려면 반드시 "IP"가 필요하다

4. TCP
   - 인터넷에서 각종 제어를 담당
   - 그 중에서 port라는 개념이 있다
   - IP는 호스트를 구별한다
   - TCP port는 호스트에서 실행되는 프로그램을 구별한다(0~65535)

5. 방화벽(firewall)
   - 가장 간단한 개념으로 허용되지 않은 포트를 모두 막는다

6. 웹
   - 인터넷 위에서, HTTP 또는 HTTP 프로토콜을 이용해 HTML 문서를 주고받는 서비스
   - HTTP는 80번, HTTPS는 443번 포트를 사용자 인증 : 신원을 확인(보통 로그인으로 서버가 사용자를 인증)
     HTTP는 사용자 측에서 서버가 인증된 서버인지 확인하는 양방향 인증을 수행
